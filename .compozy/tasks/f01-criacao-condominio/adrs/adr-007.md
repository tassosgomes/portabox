# ADR-007: Storage de Documentos via `IObjectStorage` (MinIO/S3) com Metadados em Postgres

## Status

Accepted

## Date

2026-04-17

## Context

O PRD de F01 permite upload opcional de documentos do opt-in (ata da assembleia e termo do síndico) durante o wizard. O `vision.md` já fixou a decisão de usar **MinIO em dev/local** e **S3 ou Cloudflare R2 em produção**. Falta definir como o backend integra com esses storages e como registra metadados das evidências.

## Decision

- **Abstração `IObjectStorage`** na camada de aplicação com métodos essenciais: `UploadAsync(key, stream, contentType, ct)`, `GetDownloadUrlAsync(key, ttl, ct)` (presigned URL) e `DeleteAsync(key, ct)`.
- **Adapters por ambiente** via DI: `MinioObjectStorage` em dev/local (apontando para container MinIO), `S3ObjectStorage` em produção (AWS SDK for .NET ou R2 via endpoint S3-compatible). Seleção via configuração (`Storage:Provider = Minio|S3`).
- **Credenciais e endpoints** em `IOptions<StorageOptions>` lidos de `appsettings.{env}.json` + variáveis de ambiente; nenhum secret em repositório.
- **Bucket único por ambiente** (`log-portaria-{env}`) com chaves organizadas por namespace e tenant: `condominios/{tenant_id}/opt-in/{document_id}.pdf`. Isso mantém o bucket flat mas com prefixos semanticamente úteis para ops.
- **Tabela `opt_in_document`** persiste metadados: `id` (UUID), `tenant_id`, `kind` (`Ata` | `Termo`), `storage_key`, `content_type`, `size_bytes`, `sha256`, `uploaded_at`, `uploaded_by_user_id`. **Relação 1..N com o `Condominio`** (um tenant pode anexar ata, termo ou ambos, eventualmente mais de uma versão).
- **Validação de upload**: tamanho máximo 10 MB, content-type restrito a `application/pdf`, `image/jpeg`, `image/png`. Hash SHA-256 calculado em streaming no backend para evitar carregar o arquivo inteiro em memória.
- **Acesso ao download**: somente via `GetDownloadUrlAsync` que gera presigned URL com TTL curto (5 min) — o backend nunca servir o binário diretamente. Em dev, o adapter MinIO expõe a presigned URL do próprio MinIO.
- **Antivírus / deep content inspection**: fora do escopo do MVP (Open Question).

## Alternatives Considered

### Alternative 1: Armazenar documentos como `bytea` em Postgres

- **Description**: coluna `BYTEA` na tabela `opt_in_document`, sem object storage externo.
- **Pros**: simplifica infra do MVP (sem MinIO container em dev).
- **Cons**: contraria o vision; aumenta tamanho do banco e custo de backups; Postgres não é otimizado para blobs.
- **Why rejected**: decisão arquitetural já tomada no vision + consequências operacionais.

### Alternative 2: Adiar upload para pós-MVP

- **Description**: no MVP só metadados; upload é feature de Fase 2.
- **Pros**: reduz escopo do MVP.
- **Cons**: CF3 do PRD foi escrita contemplando upload opcional; remover agora exige re-aprovação do PRD; evidência documental é desejável no piloto.
- **Why rejected**: o custo marginal de incluir via abstração `IObjectStorage` é baixo; o benefício (evidência LGPD) é alto.

## Consequences

### Positive

- Mesmo código roda em dev (MinIO) e prod (S3/R2) sem refactor.
- Binários isolados do banco transacional.
- Metadados em Postgres permitem queries rápidas sem envolver o object storage.

### Negative

- Duas fontes de verdade para documentos (DB para metadados, storage para binário) exigem cuidado em criação/deleção transacional.
- Presigned URLs precisam ser renovadas a cada request de download (baixo custo).

### Risks

- **Risco:** órfãos no object storage (upload ok, insert de metadados falha). **Mitigação:** fluxo de upload é "reservar → upload → commit"; em falha, o object key é marcado para limpeza por worker; alternativa simples: job noturno compara `opt_in_document.storage_key` com listagem do bucket.
- **Risco:** exfiltração via presigned URL compartilhada. **Mitigação:** TTL curto (5 min); log de emissão de URL com usuário requisitante.

## Implementation Notes

- Pacotes: `Minio` NuGet para adapter dev; `AWSSDK.S3` para prod/R2 (ambas S3-compatíveis).
- Hash SHA-256 calculado durante o upload em streaming; permite deduplicação futura se desejado.
- `opt_in_document.tenant_id` respeita ADR-004 (coluna + query filter).
